# Seadio Kill — 游戏设计文档 v1.0

> 以狼人杀角色为原型的卡牌对战网页游戏

---

## 1. 项目概述

**Seadio Kill** 是一款基于狼人杀角色的 1v1 卡牌对战网页游戏。玩家从共享牌库中随机抽取手牌，通过暗置角色牌、石头剪刀布争夺行动权，利用角色技能和技能牌击败对手。

### 1.1 设计目标

- **初版目标**：跑通 1v1 核心玩法，验证"暗置角色 + 石头剪刀布 + 技能牌"的组合是否有趣
- **长期目标**：扩展多人混战、阵营对抗等模式

---

## 2. 核心玩法

### 2.1 游戏流程

```
开始匹配
  → 从 16 张共享牌库中各抽 6 张手牌
  → 进入对战循环
  → 分出胜负
```

### 2.2 对战循环（每回合）

```
┌─ 回合开始 ────────────────────────────────────────────┐
│                                                        │
│  1. 暗置阶段                                           │
│     双方各从手牌中暗置（倒扣）一张牌到场上              │
│     对方不可见你出了什么牌                              │
│     场上角色存活时可跳过部署（或出双用牌当技能消耗）     │
│                                                        │
│  2. 揭示阶段                                           │
│     同时翻开，宣布回合开始                              │
│                                                        │
│  3. 石头剪刀布                                         │
│     赢的人选择以下行动之一：                            │
│       A. 攻击 → 用场上角色攻击对方                     │
│       B. 出技能牌 → 从手中打出一张技能牌               │
│       C. 换角色牌 → 替换场上角色（换下的角色保留扣血）  │
│       D. 毒药 → 女巫专属，直接击杀对方场上角色         │
│     输的人：被动应对                                    │
│                                                        │
│  4. 结算阶段                                           │
│     结算伤害，检查角色是否死亡，触发死亡技能            │
│                                                        │
│  5. 回合结束 → 进入下一回合                            │
└────────────────────────────────────────────────────────┘
```

### 2.3 胜负条件

- 对方**所有角色牌被消耗**（手牌和场上都没有角色牌）→ 你赢了
- 双方同时无法出角色牌 → 平局

### 2.4 关键机制

- **角色牌留在场上**：场上活着的角色可以一直使用，不需要每回合重新出
- **只有死亡才需要补新角色**：角色 HP 归零后被消耗，必须从手牌出新的角色牌
- **换角色有代价**：赢了回合可以换角色，但换下的角色**已扣的 HP 不恢复**
- **手牌用完就没了**：6 张牌抽完后不再补充，需要合理规划使用时机

## 3. 卡牌系统

### 3.1 卡牌类型

卡牌分为三种：**角色牌**、**技能牌**、**双用牌**。

#### 角色牌

放置到场上，拥有 HP 和 ATK，持续战斗直到死亡。纯战斗单位，无技能牌效果。

#### 技能牌

从手中打出，产生一次性效果后消耗。不能放置到场上。

#### 双用牌（角色 + 技能）

某些角色牌可以选择两种方式使用：
- **当角色牌**：上场战斗，拥有该角色的属性和被动技能
- **当技能牌**：消耗使用，产生技能牌效果，不上场

### 3.2 牌库构成（共 16 张）

| 卡牌 | 数量 | 类型 | 作为角色牌 | 作为技能牌 |
|------|------|------|-----------|-----------|
| **村民** | ×4 | 角色牌 | 2HP / 1ATK / 无技能 | — |
| **狼人** | ×4 | 角色牌 | 3HP / 1ATK / 无技能 | — |
| **猎人** | ×2 | 角色牌 | 3HP / 1.5ATK / 殉职 | — |
| **守卫** | ×2 | 双用牌 | 2HP / 1ATK | 免疫一次伤害 |
| **女巫** | ×2 | 双用牌 | 2HP / 1ATK / 复活 / 毒药 | 复活一张已失去的角色牌 |
| **预言家** | ×2 | 技能牌 | — | 查看对方所有手牌，指定其下一张出的角色 |

### 3.3 抽牌规则

- 16 张牌库洗牌后，双方各抽 **6 张**
- 剩余 4 张不使用
- **不保证**角色牌和技能牌的数量比例——可能全是角色牌，也可能技能牌居多
- 双方**看不到**对方的手牌

## 4. 角色详解

### 4.1 村民

| 属性 | 值 |
|------|-----|
| HP | 2 |
| ATK | 1 |
| 技能 | 无 |

> 基础角色，没有特殊能力，纯粹的战斗单位。

### 4.2 狼人

| 属性 | 值 |
|------|-----|
| HP | 3 |
| ATK | 1 |
| 技能 | 无 |

> 血量最高的基础角色，适合持久战。被猎人克制（猎人 1.5 伤害两回合带走）。

### 4.3 猎人

| 属性 | 值 |
|------|-----|
| HP | 3 |
| ATK | 1.5 |
| 技能 | 殉职 |

**殉职**：猎人死亡时，对对方场上角色造成 **10 点伤害**（直接击杀，可被守卫盾抵挡）。

> 攻击力 1.5，两回合攻击即可对 3HP 角色造成 3 点伤害，死后 10 点伤害直接带走一张角色牌。对狼人有天然克制。

### 4.4 守卫

| 属性 | 值 |
|------|-----|
| HP | 2 |
| ATK | 1 |
| 技能牌效果 | 免疫一次伤害 |

**当角色牌**：2HP / 1ATK，无被动技能。

**当技能牌**：消耗使用，**免疫一次伤害**（可挡普通攻击和猎人殉职，但**挡不住女巫毒药**）。

### 4.5 女巫

| 属性 | 值 |
|------|-----|
| HP | 2 |
| ATK | 1 |
| 技能 | 复活、毒药 |
| 技能牌效果 | 复活一张已失去的角色牌 |

**当角色牌**：
- **复活**：女巫死亡时，可**满血复活一次**（仅限一次）
- **毒药**：女巫在场时，可使用毒药**直接击杀对方场上一个角色**（无视守卫盾，但**预言家的躲避可以躲过**）

**当技能牌**：消耗使用，**复活一张自己已失去的角色牌**（回到手牌）。

### 4.6 预言家（纯技能牌）

| 属性 | 值 |
|------|-----|
| 类型 | 纯技能牌（不能上场） |
| 技能牌效果 | 天眼 |

**天眼**：消耗使用，**查看对方所有手牌**，并**指定对方下一张必须出的角色**。

> 预言家是唯一的纯技能牌，没有场上存在感，只能在行动阶段使用。使用后从手牌中消耗。

## 5. 石头剪刀布机制

### 5.1 规则

- 每回合双方**同时选择**石头/剪刀/布
- 赢的一方获得**本轮行动权**
- 平局 → 重新选择，直到分出胜负

### 5.2 交互流程

```
双方看到 "请选择：✊ ✌️ ✋"
  → 倒计时 3 秒
  → 双方同时揭晓
  → 赢的人选择：攻击 / 出技能牌 / 换角色牌
```

### 5.3 行动权选项

赢得石头剪刀布后，选择以下行动之一：

| 选项 | 条件 | 说明 |
|------|------|------|
| **攻击** | 场上有角色 | 用场上角色的 ATK 攻击对方场上角色 |
| **出技能牌** | 手中有技能牌 | 从手中打出一张技能牌（含双用牌的技能牌效果） |
| **换角色牌** | 手中有角色牌 | 将场上角色替换为手中另一张角色牌（被换下的角色保留已扣 HP） |
| **毒药** | 女巫在场上 | 直接击杀对方场上一个角色（无视守卫盾） |

## 6. 技术方案

### 6.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | HTML + CSS + JavaScript | 纯前端，无框架 |
| **后端** | Node.js + Express | 游戏服务器 |
| **实时通信** | Socket.io | 双向实时通信 |
| **代码格式化** | Prettier | 保持代码风格一致 |

### 6.2 项目结构

```
Seadio Kill/
├── docs/                        # 文档
│   ├── 2026-06-06-seadio-kill-v1-design.md
│   └── plans/
├── client/                      # 前端
│   ├── index.html               # 主页面
│   ├── css/
│   │   └── style.css            # 样式
│   └── js/
│       ├── main.js              # 入口 + 事件绑定
│       ├── game.js              # 客户端游戏状态
│       ├── ui.js                # DOM 渲染
│       ├── socket.js            # Socket 通信封装
│       └── data.js              # 角色/卡牌数据
├── server/                      # 后端
│   ├── index.js                 # 服务器入口 + 游戏事件
│   ├── room.js                  # 游戏房间 + 匹配系统
│   ├── logic.js                 # 战斗逻辑 + 技能效果
│   └── data.js                  # 牌库定义
├── package.json                 # 依赖配置
└── README.md                    # 项目说明
```

### 6.3 通信协议（Socket.io 事件）

#### 客户端 → 服务器

| 事件 | 数据 | 说明 |
|------|------|------|
| `join_queue` | — | 加入匹配队列 |
| `create_room` | — | 创建房间 |
| `join_room` | `{ roomCode }` | 加入指定房间 |
| `cancel_room` | — | 取消创建的房间 |
| `deploy_card` | `{ cardId }` | 暗置一张牌到场上 |
| `skip_deploy` | — | 跳过部署 |
| `rps_choice` | `{ choice }` | 石头剪刀布选择 |
| `action` | `{ type, cardId? }` | 行动选择（attack/skill/swap） |
| `witch_poison` | — | 女巫毒药（击杀对方场上角色） |
| `select_revive_target` | `{ cardId, targetCardId }` | 选择复活目标 |

#### 服务器 → 客户端

| 事件 | 数据 | 说明 |
|------|------|------|
| `game_start` | `{ roomId, yourHand, opponentHandCount, deck }` | 游戏开始 |
| `room_created` | `{ roomCode }` | 房间已创建 |
| `room_joined` | `{ roomId }` | 加入房间成功 |
| `room_cancelled` | — | 房间已取消 |
| `round_start` | `{ round, phase }` | 新回合开始 |
| `deploy_success` | `{ cardId }` | 部署成功（null 表示跳过） |
| `cards_revealed` | `{ yourCard, opponentCard }` | 双方暗置牌揭示 |
| `rps_start` | `{ message }` | 石头剪刀布开始 |
| `rps_result` | `{ result, winner?, loser? }` | 石头剪刀布结果 |
| `your_turn` | `{ actions, message }` | 轮到你行动 |
| `opponent_turn` | `{ message }` | 对方行动中 |
| `action_result` | `{ type, result, ... }` | 行动结算结果 |
| `death_skill` | `{ playerId, skill, message, ... }` | 死亡技能触发 |
| `select_revive_target` | `{ lost, cardId }` | 选择复活目标 |
| `game_over` | `{ winner, reason }` | 游戏结束 |
| `error_msg` | `{ message }` | 错误提示 |

## 8. 克制关系图

```
        猎人 ──克制──→ 狼人
         │               │
      殉职10点          3HP高血量
         │               │
         ▼               ▼
    守卫盾可挡       女巫毒药可秒
         │               │
         ▼               ▼
      守卫 ◄──克制── 女巫
                       │
                  毒药无视守卫盾
```

**简单总结**：
- 猎人克狼人（1.5ATK + 死亡10点伤害带走高血量角色）
- 女巫克守卫（毒药无视盾）
- 守卫克猎人（盾可挡殉职伤害）
- 预言家：情报型角色，查看对手手牌并指定出牌，无直接克制关系

## 9. MVP 范围（v1.0）

### ✅ 已实现

- [x] 1v1 在线匹配 + 创建/加入房间
- [x] 16 张共享牌库，每人抽 6 张
- [x] 暗置角色牌 + 揭示机制
- [x] 石头剪刀布决定行动权
- [x] 行动四选一（攻击 / 出技能牌 / 换角色 / 毒药）
- [x] 跳过部署（场上角色存活时）
- [x] 6 个角色全部实装（村民、狼人、猎人、守卫、女巫、预言家）
- [x] 所有被动技能和技能牌效果
- [x] 游戏胜负判定
- [x] 基础 UI 界面 + 牌库说明面板

### ❌ v1.0 不做

- 多人混战模式
- 阵营对抗模式
- 排位/积分系统
- 角色选择界面（随机分配）
- 自定义卡组
- 音效/音乐
- 移动端适配（先做桌面端）
